# 바이브 코딩 워크플로우 (Vibe Coding Workflow)

## 4.1 BAAS 개발 및 사용 플로우
- **Describe**: 자연어로 책의 컨셉과 요구사항 정의.
- **Plan**: Outline Planner가 챕터 구성을 제안.
- **Scaffold**: 제안된 구조를 코어 데이터 모델로 변환.
- **Refine**: 에이전트 집필 및 비평 루프 실행.
- **Finalize**: 결과물 검토 및 최종 Export.

## 4.2 특징
- "Vibe" (저자의 의도와 톤)가 전체 생성 프로세스에 일관되게 적용됨.
- 에이전트가 개발자이자 보조 작가로서 동작하는 인터랙티브 환경.
